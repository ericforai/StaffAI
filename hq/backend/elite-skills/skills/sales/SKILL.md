# 销售能力体系

> 基于契约锁销售经验总结

## Overview

销售方法论是一套完整的电子签章销售体系，核心是**销售闭环六阶段**配合**销售四力**，通过结构化的方式完成从线索到续费的全流程。

## When to Use

**Use when:**
- 销售推进遇到瓶颈，不知道下一步怎么走
- 客户在价格谈判中反复压价
- 演示不知道讲什么、怎么讲
- 客户犹豫不决，不知道怎么逼单
- 续费时客户说要考虑
- 不知道如何挖掘客户需求

**Don't use when:**
- 客户需求完全不匹配产品能力（应转线下或放弃）
- 客户已经有明确竞品在做深度对比（需要更专业的方案团队）

## 销售闭环六阶段

| 阶段 | 核心任务 | 关键动作 |
|------|----------|----------|
| 1. 基础了解 | 精准定位客户 | 线索挖掘→判断有效性→首次拜访 |
| 2. 需求价值 | 传递产品价值 | 痛点挖掘→产品介绍→方案呈现 |
| 3. 演示阶段 | 建立信任 | 三段论（挖坑→埋土→种花）、差异化 |
| 4. 认证试用 | 推动体验 | 指导配置、行业最佳实践 |
| 5. 商务谈判 | 达成合作 | 三档报价、赠券策略、底线不破 |
| 6. 服务上线 | 交付价值 | 启动会→配置→培训→验收 |

## 销售四力

| 能力 | 说明 | 提升方式 |
|------|------|----------|
| 挖掘力 | 发现客户需求的能力 | 产品学习、案例学习、行业知识 |
| 沟通力 | 传递价值、建立信任 | 顾问式提问、场景化讲解 |
| 谈判力 | 商务博弈、促成成交 | 价格策略、赠券技巧、以退为进 |
| 服务力 | 持续经营、续费增长 | 客户关怀、需求响应、价值挖掘 |

## 核心谈判技巧

| 技巧 | 应用场景 | 话术 |
|------|----------|------|
| 甩锅法 | 价格太低 | "价格比较低，商务他不批" |
| 降价不过两 | 不能一直让价 | 第一次正常沟通，第二次领导特批 |
| 赠券用起来 | 单价不降，赠券变相降价 | "同样价格可以多用XX份" |
| 三档报价 | 主推第三档 | "红花需要绿叶配" |
| 以退为进 | 站在客户角度给建议 | "低价无利润，服务跟不上" |
| 预设埋点 | 不可一降再降 | 单价是心理防线 |

## 演示三段论

**"挖坑→埋土→种花"**

1. **挖坑**：结合客户场景和痛点，吸引注意
2. **埋土**：告知如何解决，让客户放心
3. **种花**：价值升华，带来额外惊喜

## 客户画像五要素

| 判断因素 | 问题 |
|----------|------|
| 需求场景 | 数字身份/电子签章/印章管控/数据存证？ |
| 使用量 | 几份/百十份/千把份/万把份？ |
| 预算 | 有没有预算，匹配吗？ |
| 使用时间 | 什么时候启用？ |
| 决策链 | 是决策人吗？流程是什么？ |

## 产品学习四步法

1. **定位功能** — 清晰每个功能点位置
2. **理解作用** — 知道能解决什么问题
3. **掌握逻辑** — 理解背后业务逻辑
4. **串联功能** — 由点到线到面，构建体系

**升华：** 基于功能逻辑提出产品建议，展现专业性

## 销售漏斗分类

| 状态 | 条件 | 维护周期 |
|------|------|----------|
| A类 | 即将成单/明确需求 | 4天内必须维护 |
| B类 | 明确需求/短期上线计划 | 20天内必须维护 |
| C类 | 了解一下/短期无计划 | 50天内必须维护 |

## 续费沟通要点

- **服务自评**：关注使用情况、及时关怀、保持粘度
- **谈判底线**：单价不轻易降，赠券灵活运用
- **问题处理**：不惯着、不欺骗、真话不全说

## 核心心态

> **1/3是我们必须拿下来的，1/3是我们努力争取拿下来的，还有1/3是我们拿不下来的。**

> **知彼知己才能扬长避短** — 产品要专业、行业要了解、友商要熟悉

## Quick Reference

| 场景 | 应对 |
|------|------|
| 客户问最低多少钱 | "您说多少合适，我去申请？" |
| 客户一直压价 | 降价不过两次，赠券用起来 |
| 演示不知道讲什么 | 三段论 + 行业最佳实践 |
| 客户说再考虑 | 限制期限 + 同行刺激 |
| 续费客户犹豫 | 服务自评 + 价值挖掘 |

## 常见错误

**问题：** 客服角色，客户问什么答什么
**修复：** 引导客户，输出价值，让客户依赖你

**问题：** 上来就讲怎么设置
**修复：** 先讲解决方案，不讲设置

**问题：** 直接降单价
**修复：** 赠券形式降总价，不降单价

**问题：** 帮客户做认证
**修复：** 切记不要帮客户认证，让客户自己来
