# 契约锁客服服务体系

> 基于私有云项目不同阶段客服工作标准流程

## Overview

契约锁客服服务体系是一套覆盖客户全生命周期的服务标准，核心是**按项目阶段匹配服务内容**，通过标准化流程、话术规范和问题避雷方案，确保客户服务质量可控、问题可闭环。

## When to Use

**Use when:**
- 处理客户计费规则配置和预警设置
- 收到合同流程需要客服确认
- 客户验收交接阶段需要推进
- 客户提出超范围服务或费用争议
- 需要挖掘客户升级需求
- 客户欠费或维保费用拖欠

**Don't use when:**
- 客户需求完全不匹配产品能力（应转销售/项目经理）
- 涉及产品 bug 或需求（应由项目经理跟进）

## 项目阶段与客服职责对照表

| 阶段 | 客服角色 | 核心任务 |
|------|----------|----------|
| 仅计费阶段（已上线-未支持/已支持） | 仅计费客服 | 5个工作日内配置计费规则+预警 |
| 已验收-已移交 | 专属客服 | 全流程服务对接+日常维护 |
| 新购买实施中 | 仅计费客服 | 早发现、早提醒、不后置问题 |
| 已上线需支持 | 支持客服 | 协助认证/签署/证书等 |
| 验收交接阶段 | 专属客服 | 检查资料+客户验收确认 |
| 确认接收阶段 | 专属客服 | 建立企微群+明确对接流程 |
| 售后服务阶段 | 专属客服 | 高效解决问题+满意度提升 |
| 二次销售阶段 | 专属客服 | 费用结算+增值业务挖掘 |

## 计费规则标准工作流程

### 时效要求

| 事项 | 时限 | 责任人 |
|------|------|--------|
| 客户流量欠费表分发 | 每月第2个工作日内 | 张悦 |
| 各部门经理汇总反馈 | 每月15日前 | 部门经理 |
| 分配客户后预警配置 | 10个工作日内 | 客服 |
| 合同流程/事件处理 | 5个工作日内 | 客服 |

### 配置检查清单

- [ ] 计费规则可正常配置（资源包/单价模式）
- [ ] 无不合理叠加（如资源包+单价同时存在）
- [ ] 必选项已完整购买（如短信服务）
- [ ] 数字证书：长期证书+事件型证书口径一致
- [ ] 签署份数等关键计量项统计口径明确
- [ ] 发现问题第一时间和销售沟通

### 预警配置标准

- **预警模式**：无特殊要求客户使用10%预警
- **预警人**：销售及其上级（手机号+ec人员id）
- **无销售客户**：维护区总信息

## 验收交接阶段避雷指南

### 常见退回原因（6项）

1. **未提供盖章的风险告知书**
   - 原则上必须公司盖章
   - 无公章需客户方项目负责人签字
   - 特殊情况：项目经理书面发送告知客户确认并截图附流程

2. **项目验收单未盖章或无日期**

3. **服务器资料不全**

4. **客户反馈遗留问题未解决**

5. **多次回电客户无法联系，项目和销售均不配合协调**

6. **证书未培训**

### 不影响流程确认的事项

- 未交付永久license（全款后申请）
- 无测试环境（需搭建由项目经理完成）
- 无存储和备份机制（提醒客户风险）

### 交接确认原则

1. 不卡流程，满足条件尽快移交
2. 不影响服务的资料可适当放宽，影响服务的资料无法提供需告知风险
3. 问题前置，交接时先做内部沟通
4. 有无运维费用不影响流程交接

## 常见问题避雷方案

### 费用相关争议（高频重点）

**问题表现**：合同费用约定不清晰、超量使用、未购买费用项被擅自使用

**解决步骤**：
1. 拉取客户实际使用明细账单，标注超量/未购买项及费用
2. 同步账单给销售，明确需对接客户补缴
3. 若客户拒绝，内部沟通会一事一议制定方案

### 系统使用异常（紧急处理）

**问题表现**：未收到预警通知导致系统突然无法使用

**解决步骤**：
1. 核查预警设置是否正确（含阈值、接收人、频率）
2. 若已按标准设置，留存证据排查未送达原因
3. 同步对接销售，确认是否需临时开通权限
4. 要求销售1个月内给出进展反馈

### 历史遗留问题无人跟进

**问题表现**：验收后遗留问题无人对接处理

**解决步骤**：
1. 客服牵头梳理遗留问题明细（含描述、时间、影响范围）
2. 同步给销售，要求指定专人跟进
3. 客服建立跟进台账，每周跟进进度
4. 超期未解决升级至部门负责人督办

### 客户提出超范围支持

**问题表现**：客户需求超出验收约定范围

**区分处理**：
- 简单咨询/基础指导 → 免费提供
- 需额外人力/技术资源 → 明确告知有偿，由销售协商

### 客户拒绝缴纳维保费用

**解决步骤**：
1. 提前15个工作日发送到期提醒
2. 到期后按无维保标准提供基础服务
3. 书面告知客户服务范围边界
4. 同步销售协商维保费用缴纳

### 客户欠费长期拖欠

**解决步骤**：
1. 发送正式催款函（加盖公章），标注欠费金额和截止日期
2. 销售牵头每周跟进，协商分期
3. 超截止日期按合同执行暂停系统使用
4. 长期恶意拖欠评估法律手段维权

## 挖掘客户升级线索（8法）

1. **定期推送客户期刊** — 新版本亮点功能+成功案例
2. **400满意度回访** — 分析不满意点是否升级可解决
3. **线下大会邀请** — 销售深挖需求+传递新版本优势
4. **痛点整理推送** — 主动提供新版本解决方案
5. **日常响应询问** — 主动询问体验，提及新版本优化
6. **版本升级福利** — 推送升级折扣/延保等降低门槛
7. **客户使用档案** — 记录行业/痛点/需求，定向推送
8. **咨询顺势延伸** — 讲解新版本对应优化点

## 内部协作规范

- **涉及客户沟通**：优先由销售对接，客服配合
- **费用/服务范围**：避免口头承诺，重要约定签书面协议
- **所有问题**：留存书面记录（截图/邮件/会议纪要）
- **问题跟进**：明确时间节点+责任人，建立台账闭环
