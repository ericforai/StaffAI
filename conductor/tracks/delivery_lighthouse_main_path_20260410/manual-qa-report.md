## Happy path（整故事 smoke）

建议在**真实后端 + LLM 可用**环境下由产品/交付走一次端到端（自动化无法替代）：

1. `/tasks` → 高级向导 → 输入 → 澄清 → 确认设计 → 计划 → 创建任务。  
2. 任务详情 → 执行（若策略允许）→ 执行摘要可见。  
3. 「交付产物」Tab 打开（有数据时核对展示）。  
4. 「保存为模板」→ 模板中心检索（若已接模板 API）。

**工程侧已覆盖的片段：** Playwright `tasks-workspace`、`runtime-foundation` 等对列表、创建、执行、审批、错误条等有回归；**不替代**完整 intent→LLM 链路的 Happy path。

## 两种失败恢复（spec 要求 + 验证方式）

| # | 场景 | 验证方式 | 自动化 |
|---|------|----------|--------|
| **F1** | 任务列表 API 失败 | 手测：断网或 mock 500 → 出现「任务加载失败」+ **重试加载** | ✅ `tasks-workspace.spec.ts` |
