# 精英克隆模块设计方案

> 日期：2026-04-17
> 状态：已确认

## 1. 概述

### 核心定位
把公司各部门专家的技能通过访谈转化为标准 SKILL.md 文件，存储在共享目录，全员通过 Web 界面浏览、咨询和使用。

### 需求总结

| 问题 | 选择 |
|------|------|
| 核心目的 | 内部知识复用 |
| 技能转化方式 | 访谈提取（HR/培训/架构师访谈专家） |
| 使用者 | 全员可见可用 |
| 使用方式 | Web 界面咨询 |
| 审核流程 | 管理员人工审核 |
| 存储方式 | 共享目录文件存储 |
| 加载方式 | 管理员添加后自动发现 |

### 功能全景

```
精英克隆 (Elite Clone)
├── 技能浏览 - 卡片式展示所有可用技能
├── 技能详情 - 查看技能说明、使用场景、专家信息
├── 技能咨询 - AI 基于技能内容回答问题
├── 技能搜索 - 关键词搜索技能
└── 管理功能 (管理员)
    ├── 上传技能 - 上传 SKILL.md 文件
    ├── 联网搜索 - 搜索公开优质 skills 并导入
    ├── 审核发布 - 审核后发布技能
    └── 下架管理 - 管理已发布的技能
```

## 3. 数据模型

```typescript
interface EliteSkill {
  id: string;              // 唯一标识 (slug)
  name: string;            // 技能名称
  description: string;     // 一句话描述
  version: string;         // 版本号
  expert: {
    name: string;          // 专家姓名
    department: string;    // 所属部门
    title: string;         // 职位
  };
  category: string;        // 技能分类 (sales/marketing/engineering...)
  tags: string[];          // 标签
  content: string;         // SKILL.md 完整内容
  status: 'pending' | 'published' | 'deprecated';
  installCount: number;    // 安装量
  createdAt: string;
  updatedAt: string;
  createdBy: string;       // 创建人 (admin)
}
```

## 5. API 设计

| 方法 | 路径 | 功能 | 权限 |
|------|------|------|------|
| GET | `/api/elite/skills` | 列出所有已发布技能 | 公开 |
| GET | `/api/elite/skills/:id` | 获取技能详情 | 公开 |
| GET | `/api/elite/skills/:id/content` | 获取 SKILL.md 内容 | 公开 |
| POST | `/api/elite/skills` | 创建技能(上传/导入) | Admin |
| PUT | `/api/elite/skills/:id` | 更新技能 | Admin |
| DELETE | `/api/elite/skills/:id` | 下架技能 | Admin |
| POST | `/api/elite/skills/:id/publish` | 发布技能 | Admin |
| POST | `/api/elite/skills/:id/consult` | 咨询技能 (AI 回复) | 公开 |
| GET | `/api/elite/skills/search` | 联网搜索公开技能 | Admin |

## 7. 页面详细设计

### 7.1 技能列表页 (`/elite`)

**布局：**
- 顶部导航 + 搜索框
- 分类筛选标签（全部/销售/营销/工程/设计/产品/客服）
- 技能卡片网格（每行 3-4 张）

**卡片内容：**
- 封面色块 + 专家头像
- 技能名称
- 简短描述
- 分类标签
- 状态标签（已发布/待审核）

### 7.2 技能详情页 (`/elite/[skillId]`)

**布局：**
- 返回导航
- 左侧主内容区
- 右侧固定操作区

**左侧内容：**
- 技能介绍
- 核心内容摘要
- 使用场景列表
- 专家信息卡片

**右侧操作区：**
- 技能封面图
- 「咨询技能」按钮（主 CTA）
- 「预览内容」按钮
- 基础信息（元数据）

### 7.3 咨询对话框 (`/elite/[skillId]/chat`)

**布局：**
- 顶部：技能名称 + 简介/内容切换按钮
- 中间：消息列表区域（可滚动）
- 底部：输入框 + 发送按钮
- 右侧边栏：技能核心内容 + 推荐问题 + 使用技巧

**消息样式：**
- 用户消息：灰色气泡，右对齐
- AI 回复：主题色气泡，左对齐，支持 markdown
- 回复底部显示参考来源

**交互：**
- AI 回复时显示思考动画
- 推荐问题点击即可发送
- 输入框支持回车发送

### 7.4 管理面板 (`/elite/admin`)

**布局：**
- 顶部操作栏：上传按钮 + 联网搜索按钮
- 状态筛选标签
- 技能列表表格

**表格列：**
- 技能名称 + 版本
- 专家
- 分类
- 状态
- 更新时间
- 操作（审核/发布/下架）

## 8. 业务流程

### 8.1 技能创建流程

```
专家访谈 → 撰写 SKILL.md → 管理员上传 → 审核 → 发布 → 全员可见
```

1. **访谈**：HR/培训/架构师访谈专家，提炼技能
2. **撰写**：按 SKILL.md 模板撰写技能文档
3. **上传**：管理员上传文件到共享目录
4. **审核**：管理员审核技能内容
5. **发布**：审核通过后发布
6. **使用**：全员可见，可安装或咨询

### 8.2 技能咨询流程

```
用户选择技能 → 进入咨询页 → 输入问题 → AI 基于技能内容回复
```

1. 用户在详情页点击「咨询」
2. 进入咨询对话框
3. 输入问题或点击推荐问题
4. AI 读取技能内容，生成回答
5. 回复标注参考来源

## 9. 技术实现要点

### 9.1 咨询功能实现

- 用户发起咨询时，后端读取该技能的 `SKILL.md` 内容
- 将技能内容作为上下文注入 AI 对话
- AI 基于技能内容回答用户问题
- 回复需标注参考了技能的哪些部分

### 9.2 共享目录集成

- 使用 StaffAI 现有的 `skill-scanner.ts` 扫描共享目录
- 管理员上传后触发扫描更新
- 技能文件使用标准 SKILL.md 格式

### 9.3 前端技术栈

- 复用 StaffAI 前端技术栈（Next.js）
- 页面放在 `/app/elite/` 目录下
- 复用现有组件库和样式
