# StaffAI 自进化员工系统设计全案 (Project "Continuum")

## 1. 核心愿景：从“提示词工具”到“数字员工”
打破大模型“单次对话、空杯心态”的局限，让 Agent 成为具备 **长期记忆 (L3 Memory)**、**目标感 (OKR Driven)** 和 **主观能动性 (Proactive Intents)** 的虚拟团队成员。实现“越用越懂你，越用越专业”的进化闭环。

---

## 2. 核心架构：L3 专属记忆矩阵 (The Dossier)
每个 Agent 拥有一个物理隔离的 `L3_Memory` 存储块，与 L1（全局知识）和 L2（项目上下文）构成三层记忆体系。

### 2.1 记忆组成
*   **经验日志 (Experience Log)**：记录“踩过的坑”和“成功的路径”。
*   **行为偏好 (Behavioral Heuristics)**：从用户的 Code Review 和审批反馈中学习到的特定工程规范。
*   **组织认知 (Organizational Awareness)**：对项目内其他 Agent 和人类成员的协作评价。

### 2.2 反射循环逻辑 (The Reflection Loop)
1.  **任务终点 = 进化起点**：每个 Task 完成后，系统自动触发 `reflect` 动作。
2.  **特征提取**：比对“初始 Spec”与“最终产物”，通过 LLM 提取用户纠正的模式（如：必须使用异步接口、必须包含日志追踪）。
3.  **记忆注入**：将这些特征存入 L3。在下一次执行任务时，系统自动检索 L3 记忆并作为 `Private Instructions` 强制注入。

---

## 3. 驱动引擎：OKR 任务分形 (The Mission Control)
改变单纯的“人派活”模式，引入 OKR 驱动的自主循环。

### 3.1 OKR 驱动流
*   **目标设定**：用户设定长期 OKR（如：将核心模块测试覆盖率提升至 95%）。
*   **自主感知**：Agent（巡检形态）常驻后台，定期扫描代码库指标。
*   **自主提案 (Proactive Intents)**：Agent 发现缺口后，在 **Unified Task Center** 主动生成一个 `RequirementDraft` (Intent)，提示用户：“**[质检 Agent] 提议**：发现 Auth 模块覆盖率仅 60%，建议新增 12 个测试用例。查看计划？”

### 3.2 自治边界控制 (HITL Checkpoint C)
*   **计划审批**：Agent 的自主提案必须经过人审批。
*   **结果确认**：执行完毕后，由人进行最终合并确认。
*   **信任等级 (Mocked)**：在前端展示 Agent 从“新手”到“完全自治”的信任演进过程。

---

## 4. 产品表现层 (UI/UX) - 进化的可视化

### 4.1 Agent 成长面板
*   **成长时间轴**：可视化展示 Agent 学习到的关键知识点（如：“3月31日：学会了本项目特定的异常处理规范”）。
*   **记忆热力图**：展示 Agent 在不同模块积累的“经验值”。
*   **信任度指示器**：Mock 展示 Agent 的自治权限等级。

### 4.2 智能指挥舱
*   **提案中心**：集中展示由各个 Agent 主动发起的“改进建议”。
*   **OKR 达成树**：实时展示 Agent 自主任务对总体 OKR 的贡献进度。
